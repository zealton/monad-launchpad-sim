机制分析：Bonding Curve + 地板价循环贷
▎优势
1. 任意 Token 即借即贷 — 每个通过 bonding curve 发行的 token 天然自带借贷能力，无需审核上架、无需投票治理、无需配置预言机。发 token = 自动有交易 + 保底价 + 借贷。
2. 零清算风险 — 借款按 floor price 估值，floor price 只升不降，抵押物估值永远 ≥ 借款额，天然不可能被清算。对比 Aave/Compound 用 oracle 喂价，价格波动就爆仓。
3. 无需外部预言机 — 地板价由合约内部 MFR 机制计算，不依赖 Chainlink 等外部喂价。没有预言机攻击、喂价延迟、额外 gas 成本。
4. 抵押释放流动性 — 抵押的 token 不可能被卖出赎回，所以它们对应的 floor support 被释放出来可供借贷。抵押越多 → 可借额度越大，形成正向循环。
5. 买入正反馈飞轮 — 买入 → 价格涨 → MFR 触发 → FP 涨 → 借贷容量增加 → 可借更多 MON → 继续买入。激励长期持有。
6. 统一货币层 — 所有 token 借出的都是 MON，不需要跨 token 流动性匹配。MON 作为 Monad 原生资产，借出后可自由使用（买其他 token、提供流动性、链上交互等）。
7. 软锁仓减抛压 — 抵押 token 借了 MON 后需要还款才能取回，形成自然留存，减少市场抛压。
▎劣势
1. 资本效率受限于 FP/MP 比值 — 按 floor price 借贷，FP 通常是市价的 20-40%（取决于 MFR 参数）。100 万市值的 token 可能只能借 20-40 万 MON。Loop 天然收敛。
2. 借贷影响卖出流动性 — 借贷从交易池抽取 MON（抵押 token 释放的 floor support 部分）。高 util% 时，大额卖出可能因池子 MON 不够而被限额。
3. 群体正反馈过热 — 多个用户同时循环贷形成群体飞轮：A 的 Loop 推高价格 → B 的借贷容量增加也 Loop → 价格偏离内在价值。买盘停止后地板价不降，形成"虚高"局面。
4. 地板价是"名义保底" — price ≥ fp 永远成立，但大额卖出沿曲线下滑有巨大滑点。只有最后一个 token 的边际价 = 显示的 market price。
5. MC 虚高 — MC = 当前价 × 全部供应量，但早期 token 是低价买入的。bonding curve 天然 MC ≈ 2×TVL，看起来很大但实际流动性没那么多。
6. 无利息则无偿还动力 — 如果不收借贷利息，用户可以无限期占用池子流动性，没有还款紧迫感。需要设计利率机制（如基于 util% 的动态利率）来平衡。
